# Cognitive Load Assessment

Kognitive Last ist die gesamte geistige Anstrengung, ein Interface zu nutzen.
Überlastete Nutzer machen Fehler, werden frustriert und brechen ab.

## 8-Punkte-Checkliste

Prüfe jeden Punkt und zähle die nicht erfüllten (`failedCount` 0–8):

1. **Single Focus**: Lässt sich die Hauptaufgabe ohne Konkurrenzelemente abschließen?
2. **Chunking**: Informationen in greifbaren Gruppen (≤4 Elemente)?
3. **Grouping**: Verwandte Elemente visuell gruppiert?
4. **Visuelle Hierarchie**: Ist sofort klar, was am wichtigsten ist?
5. **Eine Sache nach der anderen**: Eine Entscheidung vor der nächsten?
6. **Minimale Auswahl**: ≤4 sichtbare Optionen an jedem Entscheidungspunkt?
7. **Arbeitsgedächtnis**: Muss sich der Nutzer Infos einer vorherigen Seite merken?
8. **Progressive Offenlegung**: Komplexität erst sichtbar, wenn nötig?

## Level

- 0–1 nicht erfüllt → `low`
- 2–3 → `moderate`
- 4+ → `high`

## Working-Memory-Regel

Menschen halten ≤4 Items im Arbeitsgedächtnis. Zähle an jedem Entscheidungspunkt
die gleichzeitig zu berücksichtigenden Optionen: ≤4 ok, 5–7 grenzwertig, 8+
überladen.

## Häufige Verstöße

- Optionsmauer (10+ Wahlmöglichkeiten ohne Hierarchie)
- Memory Bridge (Info aus Schritt 1 für Schritt 3 nötig)
- Versteckte Navigation (mentale Karte nötig)
- Jargon-Barriere (Übersetzung nötig)
- Visuelle Gleichgewicht (alles gleich laut → nichts sticht heraus)
- Inkonsistente Patterns
- Multitask-Demand
- Context Switch zwischen Tabs/Modals
