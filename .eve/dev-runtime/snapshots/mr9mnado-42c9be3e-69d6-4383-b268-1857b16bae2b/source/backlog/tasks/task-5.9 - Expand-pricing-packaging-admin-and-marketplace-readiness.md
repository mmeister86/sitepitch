## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Turn the Post-MVP feature set into measurable packaging, admin operations, and growth assets. The PRD calls for plan-based expansion, provider cost visibility, admin controls, growth loops, marketing assets, and marketplace-readiness documentation.

Scope includes Starter/Pro/Agency/Scale plan gates, configurable limits, upgrade CTAs at relevant boundaries, provider budgets, admin emergency controls, support workflows, growth assets such as example reports/templates/cost calculator, and marketplace-readiness documentation.

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 Admin can configure plan limits for audits, batch size, team seats, white label, custom domain, integrations, API, lead search, and extra credits without extra credits unlocking plan-gated features.
- [ ] #2 Usage and provider costs are reportable by plan, workspace, provider, feature, failed audit, and batch where applicable.
- [ ] #3 Admin can inspect workspaces/users, adjust credits/limits, disable public reports, cancel batch jobs, revoke API keys/integrations, and review Eve eval results with audit logs.
- [ ] #4 Upgrade CTAs appear at relevant limits such as credits exhausted, team seats, batch limits, white label, custom domain, integrations, and API access.
- [ ] #5 Growth assets exist for example reports by industry, demo campaign, template library, cost calculator, comparison page, and at least one case-study placeholder or workflow.
- [ ] #6 Marketplace-readiness docs cover MRR/plan distribution, usage, variable costs, retention/churn, provider dependencies, Eve/eval quality, privacy/abuse concept, setup, and operating procedures.
